## 4. 数据存储详细说明

### D1: 用户库 (PostgreSQL - `users_db`)

| 表名 | 关键字段 | 说明 |
|------|----------|------|
| `t_user` | id, phone, password_hash, status, vip_level, verified | 用户账户 |
| `t_profile` | id, user_id, nickname, gender, birthday, city, bio | 个人资料 |
| `t_personality_test` | id, user_id, mbti_type, big5_scores, love_values | 性格测评 |
| `t_photo` | id, user_id, url, audit_status, face_score | 照片 |
| `t_match_preference` | id, user_id, min_age, max_age, preferred_gender | 择偶偏好 |

### D2: 匹配库 (PostgreSQL + Redis)

| 存储 | 内容 | 说明 |
|------|------|------|
| `t_match` (PG) | 匹配记录(心仪/跳过/双向匹配) | 持久化 |
| `t_daily_recommend` (PG) | 每日推荐结果缓存 | 每日更新 |
| Redis: `rec:{userId}` | 当天推荐列表 (Set) | 过期24h |
| Redis: `like:{userId}` | 当天心仪记录 | 过期24h |
| Redis: `mutual:{userId}` | 双向匹配列表 | 持久 |

### D3: 消息库 (PostgreSQL + Elasticsearch)

| 存储 | 内容 | 说明 |
|------|------|------|
| `t_chat_session` (PG) | 聊天会话 | 持久化 |
| `t_message` (PG) | 消息记录 | 持久化 |
| ES: `messages_index` | 消息全文检索 | 搜索历史 |

### D4: 审核库 (PostgreSQL)

| 表名 | 说明 |
|------|------|
| `t_audit_log` | 内容审核记录 |
| `t_risk_log` | 风控日志 |
| `t_report` | 用户举报记录 |

### D5: 订单库 (PostgreSQL)

| 表名 | 说明 |
|------|------|
| `t_vip_order` | VIP订单 |
| `t_payment_log` | 支付流水 |

## 5. 关键数据流描述

| 流程 | 数据输入 | 处理 | 数据输出 | 涉及存储 |
|------|----------|------|----------|----------|
| 注册 | 手机号+验证码+密码 | 验证码校验→创建账号 | JWT Token | D1 |
| 实名认证 | 身份证照片+活体视频 | OCR+活体检测+公安比对 | 认证结果 | D1 |
| 性格测试 | 80题答案 | AI分析→MBTI+大五人格 | 性格报告 | D1 |
| 每日推荐 | 用户ID+偏好 | 多维度匹配计算 | Top20推荐列表 | D1, D2, AI引擎 |
| 心仪操作 | fromUser+toUser | 检查双向→匹配/跳过 | 操作结果 | D2 |
| 发送消息 | 会话ID+内容 | 存储→WebSocket推送 | 实时送达 | D3 |
| AI教练 | 会话上下文 | AI分析→生成建议 | 回复选项 | AI引擎 |
| 内容审核 | 内容(文本/图片) | AI预审→人工复审 | 审核结果 | D4 |
| VIP购买 | 套餐类型+支付方式 | 创建订单→支付→开通 | VIP状态 | D5 |
