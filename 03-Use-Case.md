## 1. 参与者 (Actors)

| 参与者 | 类型 | 描述 |
|--------|------|------|
| 普通用户 (User) | 主要 | 注册并认证的寻缘者，使用匹配与社交功能 |
| VIP用户 (VIP User) | 主要 | 付费会员，继承普通用户全部权限，额外享有高级功能 |
| AI引擎 (AI Engine) | 次要 | 系统内部AI模块，负责智能匹配、内容审核、推荐生成 |
| SMS服务 (SMS Service) | 外部系统 | 第三方短信服务商，发送验证码 |
| 支付平台 (Payment) | 外部系统 | 第三方支付（微信/支付宝），处理会员付费 |
| 实名认证服务 (ID Verify) | 外部系统 | 第三方实名认证API，身份证OCR+活体检测 |
| 运营管理员 (Operator) | 主要 | 平台运营人员，管理内容和用户 |
| 系统管理员 (SysAdmin) | 主要 | 系统最高权限，配置维护 |

## 3. 用例详细描述

### UC-001: 手机号注册

| 属性 | 描述 |
|------|------|
| **用例编号** | UC-001 |
| **用例名称** | 手机号注册 |
| **参与者** | 普通用户, SMS服务 |
| **前置条件** | 用户未注册；手机号未被使用 |
| **后置条件** | 账号创建成功，进入资料完善引导 |
| **优先级** | 高 (P0) |
| **触发条件** | 用户点击"注册"按钮 |

**基本流程 (Basic Flow):**
1. 用户输入手机号
2. 系统验证手机号格式和唯一性
3. 用户点击"获取验证码"
4. 系统调用SMS服务发送6位验证码
5. 用户输入验证码
6. 系统校验验证码（有效期5分钟）
7. 用户设置密码（8-20位，含字母+数字）
8. 系统创建账号，生成唯一用户ID
9. 系统返回注册成功，跳转资料完善页

**备选流程 (Alternative Flow):**
- A1: 手机号已注册 → 提示"该手机号已注册，请直接登录"
- A2: 验证码错误 → 提示重新输入，累计3次错误需重新获取
- A3: 验证码过期 → 提示"验证码已过期，请重新获取"
- A4: 24小时内获取超5次 → 提示"今日已达上限，请明日再试"
- A5: 密码不符合规则 → 提示具体规则要求

**异常流程 (Exception Flow):**
- E1: SMS服务不可用 → 提示"验证码发送失败，请稍后重试"
- E2: 网络超时 → 提示"网络连接失败，请检查网络"

### UC-002: 实名认证

| 属性 | 描述 |
|------|------|
| **用例编号** | UC-002 |
| **用例名称** | 实名认证 |
| **参与者** | 普通用户, 实名认证服务 |
| **前置条件** | 用户已注册登录 |
| **后置条件** | 账户标记为"已认证"，解锁完整功能 |
| **优先级** | 高 (P0) |
| **触发条件** | 用户进入认证页面或系统提示认证 |

**基本流程:**
1. 用户进入实名认证页面
2. 用户上传身份证正反面照片
3. 系统调用OCR服务识别姓名、身份证号
4. 用户确认识别结果
5. 系统启动活体检测（眨眼→张嘴→点头）
6. 活体检测通过后，系统调用公安库比对接口
7. 比对通过，系统标记账户为"已认证"
8. 系统展示"认证成功"并解锁限制功能

**备选流程:**
- A1: OCR识别失败 → 允许用户手动填写后人工审核
- A2: 活体检测未通过 → 允许重试2次，总计3次后锁定24小时
- A3: 姓名与身份证号不匹配 → 提示"身份证信息不一致"

### UC-003: AI性格测试

| 属性 | 描述 |
|------|------|
| **用例编号** | UC-003 |
| **用例名称** | AI性格测试 |
| **参与者** | 普通用户, AI引擎 |
| **前置条件** | 用户已注册 |
| **后置条件** | 生成性格报告，纳入匹配算法 |
| **优先级** | 高 (P0) |
| **触发条件** | 用户主动开始测试或系统引导 |

**基本流程:**
1. 用户选择开始性格测试
2. 系统依次展示MBTI测试题（约60题）
3. 每10题可暂存进度
4. 完成MBTI后展示婚恋价值观问卷（约20题）
5. 用户提交全部答案
6. 系统调用AI引擎分析
7. AI引擎计算MBTI类型 + 大五人格维度 + 婚恋价值观分数
8. 系统生成个性化性格报告
9. 报告展示：性格类型、核心特质、婚恋建议、理想匹配类型
10. 分析结果存入用户画像

**备选流程:**
- A1: 用户中途退出 → 自动保存进度，下次继续
- A2: 用户跳过测试 → 标记测试状态为"未完成"，允许后续补做

### UC-004: AI智能匹配推荐

| 属性 | 描述 |
|------|------|
| **用例编号** | UC-004 |
| **用例名称** | AI智能匹配推荐 |
| **参与者** | 普通用户, AI引擎 |
| **前置条件** | 用户已认证，资料完成度≥80%，当日推荐配额未用完 |
| **后置条件** | 推荐列表更新 |
| **优先级** | 高 (P0) |
| **触发条件** | 用户打开首页/手动刷新 |

**基本流程:**
1. 用户打开App首页"推荐"标签
2. 系统查询用户画像和偏好设置
3. 系统调用AI匹配引擎
4. AI引擎执行多维度匹配计算：
   - 基础条件过滤（年龄+5岁，同城±50km，学历±2级）
   - 性格维度匹配（MBTI兼容度）
   - 兴趣爱好协同过滤
   - 外貌偏好匹配（可选）
   - 行为数据权重调整
5. AI引擎返回Top 20匹配结果（按匹配度降序）
6. 系统生成每位候选人的匹配解释
7. 系统渲染推荐卡片列表
8. 用户浏览推荐结果

**匹配算法公式:**
```
MatchScore = W1×BasicMatch(0.25) 
           + W2×PersonalityMatch(0.30) 
           + W3×InterestMatch(0.20) 
           + W4×AppearanceMatch(0.10) 
           + W5×BehaviorWeight(0.15)
```

**备选流程:**
- A1: 推荐不足20人 → 放宽距离和年龄条件重算
- A2: VIP用户 → 可获取30人/天
- A3: 用户设置筛选条件 → 在结果中二次过滤

### UC-005: 心仪/跳过（滑动交互）

| 属性 | 描述 |
|------|------|
| **用例编号** | UC-005 |
| **用例名称** | 心仪/跳过滑动交互 |
| **参与者** | 普通用户 |
| **前置条件** | 推荐列表中有待操作的用户卡片 |
| **后置条件** | 用户操作记录保存；若双向心仪则匹配成功 |
| **优先级** | 高 (P0) |
| **触发条件** | 用户在推荐卡片上滑动或点击按钮 |

**基本流程:**
1. 系统展示第1张推荐卡片
2. 用户向右滑动（或点击心形按钮）
3. 系统记录"心仪"操作
4. 系统检查对方是否也心仪了自己
5. 若非双向心仪 → 切换到下一张卡片
6. 若是双向心仪 → 触发匹配成功流程 (UC-006)
7. 用户向左滑动（或点击X按钮）
8. 系统记录"跳过"操作
9. 自动切换到下一张卡片

**备选流程:**
- A1: 用户5秒内点击"撤销" → 撤回上次操作
- A2: 今日滑动配额用完 → 提示"今日已看完，凌晨6点更新"
- A3: 推荐列表清空 → 提示"暂无新的推荐，请完善资料获取更多推荐"

| **触发条件** | 用户在聊天界面点击"AI建议"按钮或系统检测到用户犹豫 |

**基本流程:**

   - 幽默风趣型

   - 好奇探索型
5. 系统展示3条建议供用户选择
6. 用户点击选择或将内容编辑后发送
7. 系统记录采纳情况用于算法优化

|------|------|
| **用例编号** | UC-010 |
| **用例名称** | 用户风控管理 |
| **参与者** | 系统管理员, AI引擎 |
| **前置条件** | 系统检测到异常用户行为 |
| **后置条件** | 风险用户被标记/限制/封禁 |
| **优先级** | 高 (P0) |
| **触发条件** | 自动巡检或人工触发 |

**基本流程:**
1. AI引擎持续监控用户行为：
   - 批量心仪操作（每秒超过10次）
   - 相同内容群发（发消息给超过20人）
   - 被多次举报（≥3次）
   - 异常登录（异地/陌生设备）
2. AI判定风险等级：低/中/高
3. 低风险 → 系统警告通知
4. 中风险 → 限制部分功能（无法发消息）
5. 高风险 → 自动封禁账号
6. 系统管理员可查看风控日志，支持手动处理

## 4. 用例关系总结

| 关系类型 | 说明 | 示例 |
|----------|------|------|
| include | A必然调用B | 每日推荐 include AI匹配引擎 |
| extends | A可选扩展B | 心仪/跳过 extends 匹配成功 |
| 继承 | B继承A的用例 | VIP用户继承普通用户的所有用例 |
